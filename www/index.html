VB Scripting Expression Examples

# Set parser checkbox to VB scripting

Formatting

Formatting Numbers

Rounding a number
Round ([AREA], 1)
"12374568.2"

Formatting Currency
[INCOME] = "63500"
FormatCurrency ([INCOME])
"63,500.00"

To 2 Decimal Places
FormatNumber([POP_2015], 2)
Convert square feet to acres, and append "Acres"
Round(([Area] * 0.00002295684), 2) & " Acres"
"45.26 Acres"

Calculate a Percentage
FormatNumber([REMNANT] / [CLEARED] ) * 100 & "%"
"45%"

Formatting Text

Concatenate Fields
[REMNANT] & " " & [TYPE]
"Of Concern 12.11.13"

Concatenate a String
"Lot Plan: " & [LOTPLAN]
"Lot Plan: 154CP98453"

Stack Labels
[FIELD1] = "New"
[FIELD2] = "Line"
[FIELD1] & vbNewLine & [FIELD2]
"New
Line"

Stack Long Strings
[LANDUSE] = "Degraded Irrigated Land"
Replace ([LANDUSE], " ", vbNewLine
"Degraded
Irrigated
Land"

Change case
[CLIENT] = "John Wayne"
LCase ([CLIENT]), Returns "john wayne"
Ucase([CLIENT]), Returns "JOHN WAYNE"

Formatting Dates & Time

FormatDateTime("1/07/1999", 1)

0 Use system settings

1 Long date format
"Friday, July 01, 1999"

2 Short date format
"1/7/99"

3 Long time format
"5:36:01 PM"

4 Short time format (24 hour clock)
"17:36"

Functions

SELECT

The SELECT * FROM clause is hard coded in ArcMap which restricts the SELECT clause to return all of the columns in the table. Opening the Select by Attribute Tool SELECT * FROM name_of_feature.shp (Select all columns from name_of_feature.shp table) This restricts the use of other SQL functionality like: DISTINCT, ORDER BY, GROUP BY. These however can be used in sub queries

IN

"LOTPLAN" IN ( '21RP741159' , '51SP107866' ) returns rows with values that equal 21RP741159 and 51SP107866
NOT "LOTPLAN" IN ( '21RP741159' , '51SP107866' ) returns rows with values that are not 21RP741159 and 51SP107866
"LOTPLAN" NOT IN ( '21RP741159' , '51SP107866' ) returns rows with values that are not 21RP741159 and 51SP107866

Formatting Text

Concatenate Fields
[LONG] + ", " + [LAT]
alternative ~ better
"%s , %s" %( [LONG] , [LAT] )
alternative ~ best
'{0},{1}'.format( [LONG] , [LAT])
"149.950117, 24.969617"

Concatenate a String>
"State: " + [STATE]
alternative ~ better
"State: %s" %[STATE]
alternative ~ best
"State: {0}.format([STATE])"
"State: Victoria"

Stacked Label
[CITY] + "\n" + [STATE]
"Melbourne
Victoria"

String Case upper, lower....
[CITY].upper()
"MELBOURNE

[CITY].lower()
"melbourne

[CITY].title()
"Melbourne

Replace string
[STATE] = "Victoria"
[COUNTRY] = "AUS"
[STATE] + "\n" + [COUNTRY].replace("AUS", "Australia"
"Victoria
Australia"

HTML Decoration Examples

Examples of HTML Syntax to decorate labels in ArcMap

Bold      not so BEAUTIFUL

"<BOL>" & [FIELD] & "</BOL>"

Italic   Just like this

"<ITA>" & [FIELD] & "</ITA>"

Underline   downunder

"<UND>" & [FIELD] & "</UND>"

All capitals   London, Munich

"<ACP>" & [FIELD] & "</ACP>"

Small capitals   Wagga Wagga, Timbuktu

"<SCP>" & [FIELD] & "</SCP>"

Superscript       Super5

"<SUP>" & [FIELD] & "</SUP>"

Subscript       Sub5

"<SUB>" & [FIELD] & "</SUB>"

Colour (RGB)

# Hint use single quotes around the RGB colour value
# You can use any value colour combination red, blue, green = {0-255}
"<CLR red = '255'>" & [FIELD] & "</CLR>"
"<CLR blue = '55'>" & [FIELD] & "</CLR>"
"<CLR green = '100'>" & [FIELD] & "</CLR>"

Colour (CMYK)

# Hint use single quotes around the CMYK colour value
# You can use any value colour combination cyan, magenta, yellow, black = {1-100}
"<CLR magenta = '100'>" & [FIELD] & "</CLR>"
"<CLR cyan = '50'>" & [FIELD] & "</CLR>"
"<CLR yellow = '75'>" & [FIELD] & "</CLR>"
"<CLR black = '80'>" & [FIELD] & "</CLR>"

Character Spacing        SubStretching

# Spacing is expressed as a percentage value to the default. 0% means no spacing adjustment.
# spacing = {1-}
<CHR> spacing = "300" Some Text <\CHR>

Character Width       Wide Load

# May not be noticable on some mobile devices.
Or with some Fonts. It works in ArcMap :-)
<CHR> width = "100" Wide load <\CHR>
